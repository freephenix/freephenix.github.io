LeetCode-328. 奇偶链表

题目

给定单链表的头节点 head ，将所有索引为奇数的节点和索引为偶数的节点分别组合在一起，然后返回重新排序的列表。

第一个节点的索引被认为是 奇数 ， 第二个节点的索引为 偶数 ，以此类推。

请注意，偶数组和奇数组内部的相对顺序应该与输入时保持一致。

你必须在 O(1) 的额外空间复杂度和 O(n) 的时间复杂度下解决这个问题。

示例 1:

输入: head = [1,2,3,4,5]
输出: [1,3,5,2,4]

示例 2:

输入: head = [2,1,3,5,6,4,7]
输出: [2,3,6,7,1,5,4]

提示:

  • n == 链表中的节点数
  • 0 <= n <= 10^4
  • -10^6 <= Node.val <= 10^6

思路

这道题其实是考验大家对于链表这种数据结构的熟练程度的。

题干中直接说明了时间、空间复杂度要求，那就必须得遵守了。

我最开始的思路就是假定奇偶两个链表头指针，然后再起奇偶两个now指针, 一个 计次变量 来一步步往两个链表扔。

但是提交之后发现在有些用例里依然会被判内存超出，那就说明变量还是太多了，得砍。

然后想到了下面几个简化点：

  • head 就是奇数链头指针，可以省去一个变量
  • 偶数链头指针 省不掉的
  • 两个 now指针 怎么想感觉也去不掉，那就尝试去掉 计次变量
  • 偶数链 now 指针不再只承载连接节点的功能，而是跟奇数链now指针配合，交替拆走 偶数节点，这样就可以去掉 计次变量 了

大体思路就是这样，说起来抽象，往下去看代码会比较简单

去看官方题解时候发现思路是不谋而合的，不过看到有个评论差点笑喷，人才太多了，真的：

代码

func oddEvenList(head *ListNode) *ListNode {
	if head == nil || head.Next == nil {
		return head
	}
    // 暂存需要拼接的偶数链头部
    eventHead := head.Next
    // 暂存需要拼接的偶数链头部
	oddNow, evenNow := head, head.Next
    // 看是不是还有需要往奇数链放置的节点（evenNow.Next）
	for evenNow != nil && evenNow.Next != nil {
        // 奇数链先连上自己下一个节点
		oddNow.Next = evenNow.Next
		oddNow = oddNow.Next
        // 偶数链跟上连上自己的下一个节点
		evenNow.Next = oddNow.Next
		evenNow = evenNow.Next
	}
    // 拼接奇偶链
	oddNow.Next = eventHead
	return head
}
