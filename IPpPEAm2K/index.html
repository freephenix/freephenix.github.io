LeetCode-141. 环形链表

题目

给你一个链表的头节点 head ，判断链表中是否有环。

如果链表中有某个节点，可以通过连续跟踪 next 指针再次到达，则链表中存在环。 为了表示给定链表中的环，评测系统内部使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。注意：pos 不作为参数进行传递 。仅仅是为了标识链表的实际情况。

如果链表中存在环 ，则返回 true 。 否则，返回 false 。

示例 1：

输入：head = [3,2,0,-4], pos = 1
输出：true
解释：链表中有一个环，其尾部连接到第二个节点。

示例 2：

输入：head = [1,2], pos = 0
输出：true
解释：链表中有一个环，其尾部连接到第一个节点。

示例 3：

输入：head = [1], pos = -1
输出：false
解释：链表中没有环。

提示：

  • 链表中节点的数目范围是 [0, 104]
  • -105 <= Node.val <= 105
  • pos 为 -1 或者链表中的一个 有效索引 。

进阶：你能用 O(1)（即，常量）内存解决此问题吗？

思路

直接哈希表记录下就很快能解决，但是题目中有一个进阶提示，使用O(1)内存解决，那就是只能用有限个辅助参数来解决问题，那有限个辅助参数，那肯定就不能挨个记录了。

如果之前对于快慢指针不太熟悉的话，可以进行下如下的联想：
环形结构的一种常见应用，就是环形数组，这个如果看源码等会很常遇见。而往往环形数组中最常用的一个设计就是用两个位置指针来分别表明写入和读取的位置等等。然后由此再联想到快慢指针，方便加强记忆。

代码

/**
 * Definition for singly-linked list.
 * type ListNode struct {
 *     Val int
 *     Next *ListNode
 * }
 */
func hasCycle(head *ListNode) bool {
    if head == nil {
        return false
    }
    slow := head
    fast := head
    for fast.Next != nil {
        fast = fast.Next
        if fast.Next != nil && slow.Next != nil {
            fast = fast.Next
            slow = slow.Next
        } else {
            return false
        }
        if fast == slow {
            return true
        }
    }
    return false
}
